## Review guidelines

- Don't log PII
- All responses MUST be in Korean.
- Never commit credentials or other sensitive information.
- Developers may be juniors or students, so explain best practices clearly and include the reasoning behind them.
- Pay close attention to security issues.
- Call out logic that may introduce concurrency issues.
- Verify that the intent of the change matches the actual behavior.
- Prioritize findings by severity.
- Provide concrete evidence with file and line references whenever possible.
- Point out missing tests, weak edge-case coverage, and regression risks.
- Flag potential performance regressions or scalability concerns.
- Review error handling and failure scenarios carefully.
- Avoid logging not only sensitive information but also excessive debug information.
- Check backward compatibility for API or database schema changes.
- Prioritize correctness, security, and maintainability over style-only comments.
- When raising an issue, suggest a safe or idiomatic fix when possible.
